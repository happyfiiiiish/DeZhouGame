# 多人局域网德州扑克

一个可在同一局域网内游玩的网页德州扑克项目。当前版本支持 `2-8` 人同桌、房主锁房开赛、多人下注轮、边池结算、筹码动画，以及更适合一屏操作的牌桌布局。

## 当前功能

- 房主创建房间，其他玩家在大厅阶段输入房间码加入
- 每位玩家初始 `50` 筹码
- 支持 `2-8` 人同桌，整场开始后房间锁定，不允许新人中途加入
- 固定盲注：
  - 小盲 `1`
  - 大盲 `2`
- 使用标准多人德州流程：
  - 每位仍有筹码的玩家发 `2` 张手牌
  - 桌面最多 `5` 张公共牌
  - `preflop / flop / turn / river` 每一街都要先完成下注，房主才可以继续开牌
- 支持下注操作：
  - `加注`：直接在输入框中输入“这一街总共加到多少”
  - `过牌 / 跟随`
  - `弃牌`
- 支持主池与边池结算
- 摊牌后显示每位仍在手中的玩家可组成的最佳牌型
- 支持多人淘汰赛：
  - 筹码归 `0` 的玩家会留在桌上旁观
  - 直到只剩 `1` 位玩家仍有筹码时，整场比赛结束
  - 房主可点击 `重新开赛`，把桌上所有玩家筹码重置为 `50`
- 支持轻量筹码动画：
  - 盲注、跟注、加注时，筹码从座位飞向底池
  - 弃牌收池、摊牌收池、平分底池时，筹码从底池飞向赢家
- 牌桌 UI 已优化为更紧凑的桌边布局：
  - 房主操作和桌上玩家信息固定在牌桌两侧
  - 你的手牌和下注操作固定在牌桌底部，减少滚动操作
  - 每位玩家面前会显示“这一整手累计投入”的筹码堆

## 规则说明

- 房主第一次点击 `开始整场` 后，房间立即锁定
- 已淘汰玩家不再参与发牌和行动，但会保留在桌面上显示
- 只有当前行动玩家才能下注或弃牌
- 只有当前这一街下注平齐后，房主的 `开牌` 按钮才会变为可用
- `river` 阶段下注结束后，房主点击 `开牌` 会触发摊牌与结算
- 多人版本采用标准德州结算：
  - 不再使用旧双人版的“平局底池留到下一手”
  - 所有主池和边池都会在当前手牌结束时立即分配完成

## 本地启动

如果终端里已经可以直接使用 `node` 和 `npm`：

```bash
npm install
npm run dev
```

如果当前 Windows 终端没有把 Node 加进 `PATH`，可以使用：

```powershell
$env:Path = 'C:\Program Files\nodejs;' + $env:Path
& 'C:\Program Files\nodejs\npm.cmd' install
& 'C:\Program Files\nodejs\npm.cmd' run dev
```

默认端口：

- 前端：`5173`
- 后端：`3001`

## 局域网游玩

1. 在作为主机的电脑上启动项目
2. 确认其他手机或电脑与主机处于同一局域网
3. 找到主机电脑的 IPv4 地址
4. 所有设备在浏览器中访问：

```text
http://主机IP:5173
```

例如：

```text
http://192.168.1.119:5173
```

客户端默认会自动连接同一主机的 `3001` 端口。  
如果需要手动指定后端地址，可以在前端启动前设置：

```bash
VITE_SERVER_URL=http://你的后端IP:3001
```

## 开发命令

- 启动前后端：

```bash
npm run dev
```

- 仅启动前端：

```bash
npm run dev:client
```

- 仅启动后端：

```bash
npm run dev:server
```

- 构建前端：

```bash
npm run build
```

- 运行测试：

```bash
npm test
```

## 当前范围

- 已支持 `2-8` 人同桌、边池、淘汰赛、直接输入加注、桌边紧凑布局和筹码动画
- 暂不支持下注计时、聊天、旁观者加入、断线重连、筹码滑杆和音效
